常见问题

OpenFang 是什么？

expand_more
OpenFang 是一个基于 Rust 构建的生产级 Agent 操作��统。与聊天机器人框架不同，它专为让 AI 智能体 7×24 小时自主运行而设计——不需要你不断发送指令。它是 AI 智能体的操作系统层，提供运行时、安全防护和渠道接入基础设施。

和 OpenClaw 有什么区别？

expand_more
OpenClaw 适合个人使用和探索性实验。OpenFang 是你需要生产级可靠性时的选择——Rust 代替 TypeScript、16 层安全防护而非基础沙箱、内置 Hand 而非拼凑胶水代码。OpenFang 还提供了一键迁移路径：openfang migrate --from openclaw，之前的记忆和 workflow 基本无损过来。

什么是 Hands？

expand_more
Hand 是 OpenFang 的自主能力单元，类似增强版插件，但完全自包含。每个 Hand 预置了专属模型、工具集和执行工作流——你不需要手动组装任何东西，激活即用。七个内置 Hand 分别覆盖：视频（Clip）、线索挖掘（Lead）、情报监控（Collector）、趋势预测（Predictor）、深度研究（Researcher）、社媒管理（Twitter）、网页自动化（Browser）。

支持哪些 LLM 提供商？

expand_more
OpenFang 支持 12 家主流 LLM 提供商，包括 Anthropic、OpenAI、Gemini、Groq、DeepSeek、Mistral、Together、Ollama、vLLM 等，共覆盖 123+ 个模型。所有提供商均需 API Key，暂不支持 OAuth 认证。每个 Hand 可独立配置使用不同的提供商。

支持哪些通讯渠道？

expand_more
OpenFang 支持 40 个渠道适配器，是目前所有 Agent 框架中覆盖最广的。包括 Telegram、Discord、Slack、WhatsApp、Signal、Matrix、Teams、Google Chat、飞书、钉钉、Mastodon、Bluesky、LinkedIn、Reddit、IRC 等 40 个平台。你团队用的渠道，OpenFang 基本都接了。

适合生产环境使用吗？

expand_more
OpenFang v0.1.0 通过了 1,767+ 项测试，零 Clippy 警告。16 层安全防护包括 WASM 沙箱、Merkle 哈希链审计追踪和 SSRF 防护。v1.0 之前 API 可能有破坏性变更，生产环境建议锁定版本。核心运行时已稳定并在实际生产环境中部署使用。
